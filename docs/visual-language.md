# Akasha visual language

## Product posture

Akasha is a calm, local-first daily command centre. The dashboard must answer three questions within five seconds: what matters now, what else is open, and how balanced is the day?

The visual idea is **bio-circuit minimalism**: precise PCB geometry softened by organic reptile forms. Cyberpunk atmosphere comes from reflected light, depth, and selective luminous signals—not from glowing every border.

## Principles

1. Calm before spectacle. Dark surfaces and whitespace carry the interface; neon identifies focus and state.
2. One current. The primary focus and Today list dominate the information hierarchy.
3. Local and private. Persistence is on-device and the interface says so plainly.
4. Reptile, not mascot. Crocodile cues appear in the mark, eye geometry, ridges, and traces rather than cartoon illustration.
5. Fast by default. Common actions are one click or one shortcut away.

Spacing follows the 4, 8, 12, 16, 24, 32, 48, 64 scale. Control, panel, and feature radii are 6, 10, and 14 pixels. Motion uses 120, 180, or 250 milliseconds and respects reduced-motion preferences.

## Typography and iconography

Use a disciplined sans-serif system stack for primary reading. Monospace is reserved for metadata, shortcuts, and machine-like status. Use Lucide icons at 15–20 pixels with labels where meaning is not universal.

## Interaction states

Controls require visible hover, active, focus-visible, and disabled states. Completed tasks retain context while becoming visually quiet. Empty task lists should feel intentional rather than broken. Akasha must remain usable at compact desktop widths and through keyboard navigation.
