# Socionics model

Canonical type IDs use uppercase MBTI-form names (`ISTP = TiSe`, `ISTJ = SiTe`). Four-letter socionics alternatives are presentation-only and lowercase their final letter (`ISTj`, `ISTp`). Temperaments, groups, and relations derive from the canonical function core, never from the alias's final letter.

Keep these rules in `frontend/src/domain/socionics/socionicsModel.ts`; React components consume domain results instead of reimplementing them.
