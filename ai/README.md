# Akasha knowledge base

Durable project knowledge by owner:

- `product/` — intent, brand, and domain model;
- `features/` — workflow behavior;
- `integrations/` — external-system contracts;
- `project/` — architecture, migrations, and legacy context;
- `decisions/` — lasting tradeoffs that need their own history.

Start with [the architecture map](project/ARCHITECTURE.md). Prefer updating a focused document over creating another.
