# Akasha knowledge base

Durable project knowledge, organized by owner:

- `product/` — product intent, brand, and the socionics model;
- `features/` — workflow boundaries and behavior contracts;
- `integrations/` — external-system contracts;
- `project/` — architecture, migrations, and legacy context;
- `decisions/` — focused decision records when a lasting tradeoff needs its own history.

Start with [the architecture map](project/ARCHITECTURE.md) for code placement and dependencies. Update an existing focused document before creating another one.
